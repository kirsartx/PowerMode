# PowerMode v1.0.0 Release Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Publish the verified `main` branch as the first stable public PowerMode release with one portable ZIP whose downloaded hash matches the locally validated artifact.

**Architecture:** Release documentation is merged before tagging so the tag identifies one immutable, documented source state. The portable archive is rebuilt from that exact state, audited locally, uploaded to a draft GitHub Release, downloaded into a separate verification directory, and published only after the two hashes match.

**Tech Stack:** Git, GitHub CLI (`gh`), PowerShell 7/Windows PowerShell, .NET 10 SDK, xUnit, existing `scripts/Publish-Portable.ps1`.

## Global Constraints

- Release tag and title are exactly `v1.0.0` and `PowerMode v1.0.0`.
- The GitHub Release is public, stable, and not a prerelease.
- Upload exactly one binary asset: `PowerMode-win-x64.zip`.
- Do not upload a loose EXE, source archive replacement, installer, or symbols.
- Do not add GitHub Actions, an installer, signing, auto-update, UI changes, or power-management behavior.
- Never move or force-update an existing tag.
- Any failed gate stops the release before publication.
