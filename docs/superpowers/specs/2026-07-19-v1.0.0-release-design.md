# PowerMode v1.0.0 Release Design

## Goal

Publish the current `main` branch as PowerMode's first stable public release so users can download one verified portable archive without assembling source files or dependencies.

## Scope

- Create annotated tag `v1.0.0` at the verified `main` commit.
- Create a public GitHub Release named `PowerMode v1.0.0`.
- Upload only `PowerMode-win-x64.zip`.
- Include the archive's SHA-256 checksum in the release notes.
- Document system requirements, portable startup steps, major capabilities, recovery behavior, and links to the full documentation.
- Keep the existing source tree, configuration format, application behavior, and branch history unchanged.

## Exclusions

- No installer, signing certificate, Microsoft Store package, or auto-updater.
- No loose EXE upload because the application requires adjacent runtime files.
- No GitHub Actions or automatic release workflow in this iteration.
- No additional UI, UX, or power-management changes.

## Release Artifact

The artifact is produced from the exact tagged source with:

```powershell
powershell -ExecutionPolicy Bypass -File .\scripts\Publish-Portable.ps1 -Root . -CreateZip
```

Expected output:

```text
dist/
├── PowerMode-win-x64/
│   ├── App/
│   │   └── PowerMode.exe
│   ├── PowerModeSwitcher.bat
│   └── README.md
└── PowerMode-win-x64.zip
```

The release uploads the ZIP only. The SHA-256 value must be calculated after the final build and copied exactly into the release notes.

## Release Notes

The notes will be concise and bilingual-friendly, with these sections:

1. Highlights: Simple/Professional modes, recommendation, hardware capability detection, recovery center, Fluent/accessible interface, and portable CLI compatibility.
2. Requirements: Windows 10/11 x64 and administrator permission for power-plan changes.
3. Quick start: extract the complete ZIP, then run `App\PowerMode.exe`.
4. Safety: recommendations are not forced; actions are recorded and recoverable where supported.
5. Verification: Release test/build counts and SHA-256.
6. Documentation links: README, user guide, and development guide.

## Verification Gates

Before publishing:

1. Local `main` and `origin/main` resolve to the same commit.
2. Working tree is clean.
3. Full Release test suite passes with zero failures.
4. Release build completes with zero warnings and zero errors.
5. Portable archive is rebuilt from the tagged source.
6. `App\PowerMode.exe` exists and completes a responsive launch smoke test.
7. ZIP contains no `.git`, `.worktrees`, `.superpowers`, `src`, `tests`, `bin`, or `obj` paths.
8. ZIP hash and size are recorded.

After publishing:

1. Tag `v1.0.0` points to the verified `main` commit.
2. GitHub Release is public and not marked as a prerelease.
3. Exactly one binary asset is attached: `PowerMode-win-x64.zip`.
4. The downloaded asset hash matches the locally published archive.

## Failure Handling

- If any pre-publish gate fails, do not create the release.
- If the tag exists at another commit, stop instead of moving or force-updating it.
- If asset upload or release creation is incomplete, keep the release as a draft or delete only the incomplete release after confirming its state; never replace source history.
- Keep the previous local portable archive until the new archive and checksum have been verified.
