# PowerMode 成熟原生体验设计

## 目标

在现有 WinUI 3、.NET 10、便携部署和 CLI 架构上渐进式提升 PowerMode 的用户体验、稳定性与可维护性。

本次设计不重写现有模式切换链路，不引入新的生产依赖，不改变四种模式的实际电源配置，不破坏现有配置、历史、备份和便携版目录。

核心结果：

- 默认提供简洁、可理解的简单模式；
- 专业功能继续保留，但不占用简单模式主界面；
- 启动后以轻量、透明规则给出模式建议，只推荐不强制；
- 根据硬件能力隐藏或禁用无效功能；
- 将现有历史和配置备份整合为恢复中心；
- 保持 Fluent Design、系统主题、强调色和高 DPI 体验；
- 将 README 首页与完整文档分层。

## 非目标

- 不改写 `PowerModeSwitcher.bat` 的四种模式定义；
- 不引入 MVVM 框架、依赖注入容器、数据库或云服务；
- 不使用 AI、机器学习或联网服务生成建议；
- 不自动应用建议模式；
- 不重置 Windows 电源方案；
- 不新增复杂动画、首次启动向导或功能市场；
- 不拆分为两个主窗口，也不改造成 NavigationView 多页面应用。

## 架构策略

采用单窗口、状态驱动分层方案。

保留现有 `MainWindow`、`SettingsWindow`、`InsightsWindow`、`MonitoringService`、`SystemIntegrationService`、`AutomationEngine`、`HistoryStore`、`SettingsStore` 和 CLI 调用链。

新增少量职责单一的纯逻辑组件：

- `ExperienceMode`：`Simple` 或 `Professional`；
- `HardwareCapabilities`：一次启动期硬件能力快照；
- `HardwareCapabilityService`：轻量、只读、可超时的能力检测；
- `ModeRecommendation`：目标模式、原因和检测完整度；
- `ModeRecommendationService`：透明的确定性推荐规则；
- `CapabilityVisibilityPolicy`：根据体验模式和能力计算显示、禁用及原因；
- `RecoveryService`：组合现有历史、配置备份和模式切换能力；
- `RecoveryCenterWindow`：恢复中心 WinUI 窗口。

主窗口继续负责 UI 编排和调用现有模式切换入口。新增服务不直接操作 XAML 控件。

## 简单模式与专业模式

### 配置兼容

`PowerModeSettings` 新增 `ExperienceMode` 字段。

- 新安装默认 `Simple`；
- 旧配置缺少该字段时，反序列化后归一化为 `Simple`；
- 切换体验模式只保存应用设置，不触发电源模式切换；
- 导入旧配置后仍可正常使用；
- 导入包含未知体验模式值的配置时回退到 `Simple`。

### 简单模式

简单模式默认显示：

- 应用标题、最后更新时间和简单/专业切换；
- 当前状态卡片；
- 建议模式卡片、推荐原因和“一键应用”；
- 远程推荐、低功耗、平衡、高性能四个现有按钮；
- 刷新和恢复中心；
- 底部简洁状态反馈。

简单模式隐藏：

- 自动切换和实时监控快捷开关；
- 自定义 CPU；
- 校验、修复和 WiFi 操作；
- 完整日志；
- 功能中心；
- 系统洞察；
- CLI 路径。

隐藏专业区域后，布局自动收拢，不保留空白日志列。简单模式不创建第二套模式按钮或第二套切换逻辑。

### 专业模式

专业模式在简单模式能力之上显示现有全部专业入口：

- 自动切换；
- 实时监控；
- 自定义 CPU；
- 校验、修复和 WiFi 操作；
- 日志；
- 功能中心；
- 系统洞察；
- CLI 路径。

体验模式切换在当前会话立即生效，并保存为下次启动偏好。

## 启动能力检测

`HardwareCapabilityService` 在主窗口显示后异步执行，不阻塞首屏和模式按钮。

检测项目：

- 是否存在电池；
- 是否支持内部显示器亮度控制；
- 是否存在 NVIDIA 独显；
- `nvidia-smi` 是否可调用；
- 是否存在可控制的 WiFi 适配器；
- 是否存在可用温度传感器；
- 通知、全局快捷键等系统集成是否可用。

行为约束：

- 每项检测独立执行并设置短超时；
- 单项异常转换为 `Unknown` 或 `Unsupported`，不让整个检测失败；
- 检测只读，不要求管理员权限；
- 检测结果在本次进程内缓存；
- 完整诊断仍由现有洞察和诊断功能负责；
- 能力检测不写操作历史。

### 能力呈现策略

- 简单模式：不支持或未知的非核心功能直接隐藏；
- 专业模式：入口保留但禁用，并通过 ToolTip 或辅助文本解释原因；
- 核心四模式按钮不因遥测能力缺失而隐藏；
- NVIDIA 功耗不可用只影响 GPU 功耗卡片和相关洞察，不影响模式切换；
- 无电池时隐藏低电量相关设置，但保留低功耗模式；
- 无 WiFi 适配器时禁用 WiFi 操作；
- 无温度数据时禁用温度保护，并说明未检测到受支持传感器。

## 建议模式

`ModeRecommendationService` 使用本地确定性规则，不读取网络，不使用 AI，不自动应用。

输入：

- 当前供电状态；
- 电池电量；
- 配置中的远程进程列表；
- 配置中的高负载进程列表；
- 当前运行进程；
- 温度保护状态；
- 硬件能力快照。

优先级从高到低：

1. 温度保护已触发：推荐低功耗；
2. 检测到远程进程：推荐远程推荐；
3. 检测到高负载进程且未触发温度保护：推荐高性能；
4. 电池供电或电量低于配置阈值：推荐低功耗；
5. 其他插电场景：推荐平衡。

输出包含：

- 标准模式名；
- 面向用户的一句话原因；
- 检测是否完整；
- 生成时间。

推荐卡片行为：

- 用户点击“一键应用”后复用现有 `RunModeWithContextAsync`；
- 触发来源记录为 `recommendation`；
- 推荐原因写入操作历史；
- 忽略或关闭建议不产生系统修改；
- 推荐服务异常时回退为平衡，并显示“无法完成完整检测，按日常插电场景推荐”；
- 建议在供电状态、关键进程或温度保护状态发生明显变化后更新，不进行高频轮询。

## 恢复中心

新增 `RecoveryCenterWindow`，从主窗口简单模式和专业模式均可进入。

### 撤销最近模式切换

- 从历史中选择最近一次成功、目标为标准模式、尚未撤销的操作；
- 显示操作时间、触发来源、原模式、目标模式和原因；
- 撤销时复用现有模式切换入口切回 `PreviousMode`；
- 撤销记录关联原操作 ID；
- 已撤销记录不再作为下一次撤销目标；
- 撤销失败不修改原记录状态；
- 自动化、托盘、快捷键、建议卡片和手动按钮产生的成功模式切换都可撤销；
- 无有效历史时禁用撤销并说明原因。

### 恢复最近配置备份

- 复用现有配置备份列表；
- 选择与当前配置哈希不同的最近备份；
- 恢复前自动创建当前配置的安全备份；
- 使用现有原子恢复逻辑；
- 恢复成功后重新加载 `PowerModeSettings` 并刷新界面状态；
- 不改变当前 Windows 电源方案；
- 无可恢复备份时禁用入口。

### 恢复默认配置

- 操作前显示明确二次确认；
- 如果当前配置存在，先创建 `before-reset-defaults` 安全备份；
- 写入经过 `SettingsStore.Normalize` 的默认 `PowerModeSettings`；
- 默认体验模式为简单；
- 保留历史、备份、诊断和遥测文件；
- 不重置 Windows 电源方案；
- 写入失败时保持原配置，并显示可复制错误。

### 操作记录兼容

现有 `SwitchHistoryEntry` 增加可选字段：

- `OperationKind`，旧记录缺失时解释为 `mode-switch`；
- `RelatedOperationId`，用于关联撤销目标；
- `IsUndo`，标识撤销操作。

旧 JSONL 记录继续可读。新增写操作记录成功、失败、耗时和错误：

- 应用建议；
- 撤销模式切换；
- 恢复配置备份；
- 恢复默认配置。

能力检测、推荐计算、体验模式显隐属于只读行为，不写操作历史。体验模式偏好仍由配置备份体系保护。

## Fluent Design 与可访问性

- 继续使用现有 Mica、系统主题和 WinUI ThemeResource；
- 当前模式和建议卡片使用系统 Accent Brush，移除新增路径中的硬编码颜色；
- 不添加持续动画，只使用 ProgressRing、ProgressBar 和必要的显隐反馈；
- 继续支持系统高对比度、强调色、高 DPI 和布局取整；
- 窄窗口继续使用紧凑工具栏；
- 简单模式隐藏专业区域后采用单列或自适应布局；
- 专业模式保留现有双列布局；
- 体验模式、建议理由、禁用原因和恢复结果设置可访问名称与 LiveSetting；
- 所有高影响恢复操作使用 ContentDialog 确认；
- 不用颜色作为唯一状态表达。

## 数据流

启动流程：

1. 加载并归一化现有配置；
2. 显示主窗口和上次体验模式；
3. 执行现有轻量状态刷新；
4. 后台创建硬件能力快照；
5. 根据能力应用显隐策略；
6. 计算建议模式并显示原因；
7. 用户选择是否应用。

模式切换流程保持现有结构：

1. 立即激活核心 Windows 电源方案；
2. 乐观更新核心卡片；
3. 后台调用 CLI 同步细项；
4. 自动读取真实状态校准卡片；
5. 记录成功、失败和回滚信息。

恢复流程：

1. 读取可恢复目标；
2. 显示影响范围；
3. 用户确认；
4. 创建安全记录或安全备份；
5. 执行原子操作；
6. 重新加载配置或状态；
7. 写入恢复操作历史；
8. 在主窗口显示结果。

## 错误处理与降级

- 能力检测采用每项独立超时和部分成功；
- 推荐服务不抛出到 UI，失败时返回带解释的平衡建议；
- 模式应用失败继续使用现有电源方案回滚；
- 配置恢复和默认重置失败不覆盖有效配置；
- 历史文件中单行损坏继续被忽略；
- 恢复中心无法读取历史或备份时只禁用对应入口；
- 所有后台结果在应用到 UI 前检查当前请求代次，防止旧结果覆盖新状态；
- 错误信息显示简洁摘要，专业模式日志保留技术细节。

## 测试策略

新增 `.NET 10` 测试项目，仅增加测试依赖，不增加生产运行时依赖。

纯逻辑测试覆盖：

- 旧配置缺少 `ExperienceMode` 时默认为简单；
- 未知体验模式值回退为简单；
- 简单模式和专业模式的能力显隐矩阵；
- 推荐规则优先级；
- 推荐原因与目标模式一致；
- 检测部分失败时仍返回快照；
- 无电池、无 NVIDIA、无 WiFi、无温度传感器组合；
- 最近可撤销操作选择；
- 已撤销操作不重复出现；
- 配置恢复前创建安全备份；
- 默认重置保留历史与备份；
- 错误和超时降级。

集成验证覆盖：

- Release x64 编译零错误；
- 便携发布脚本与 ZIP；
- 旧配置启动；
- 简单/专业切换持久化；
- 建议只在点击后应用；
- 恢复中心三种操作；
- 系统浅色、深色、高对比度；
- 100%、125%、150% DPI；
- 窄窗口和宽窗口；
- 不支持硬件能力时的隐藏或禁用表现。
