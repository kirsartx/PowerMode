## 简单模式与专业模式

体验模式保存在 `settings.json` 的 `ExperienceMode` 字段，默认为 **Simple（简单）**。

| | 简单模式 | 专业模式 |
| --- | --- | --- |
| 四个预设按钮 | 有 | 有 |
| 建议卡片 | 有 | 有 |
| 状态卡片 | 有 | 有 |
| 恢复中心入口 | 有 | 有 |
| 语言切换 | 有 | 有 |
| 自定义远程 CPU 滑杆 | 隐藏 | 显示 |
| 校验 / 一键修复 / WiFi 相关高级操作 | 隐藏 | 显示 |
| 实时命令日志面板 | 隐藏 | 显示 |
| 顶部刷新、功能中心、洞察、自动化快切 | 隐藏 | 显示 |

能力检测会进一步隐藏或禁用不受支持的控件（例如无电池、无可用 NVIDIA 遥测、无 WiFi 适配器时），并给出原因提示；不会伪造硬件读数。

主窗口右上角可在简单 / 专业之间切换，选择会写入配置并在下次启动保留。

## 启动方式

### 便携包

构建产物目录（或解压 ZIP 后）：

```text
dist\PowerMode-win-x64\
```

双击：

```text
00-START PowerMode.bat
```

该启动器会启动同目录下的 `App\PowerMode.exe`。请保持整个目录结构，不要只复制 `App\PowerMode.exe`。

可选 ZIP：`dist\PowerMode-win-x64.zip`（发布脚本带 `-CreateZip` 时生成）。

### 源码树兼容入口

| 文件 | 行为 |
| --- | --- |
| `PowerModeWinUI.bat` | 调用 `scripts\Launch-PowerMode.bat`：若便携版过期或不存在则先 `Publish-Portable.ps1`，再启动 `dist\...\App\PowerMode.exe` |
| `PowerModeSwitcher.bat` | 转发到 `src\PowerMode.Cli\PowerModeSwitcher.bat` |
| `Build-Portable.bat` | 转发到 `scripts\Build-Portable.bat` / 发布脚本 |

PowerMode 使用单实例互斥：重复启动会唤醒已有窗口，不会创建第二套托盘、热键或自动化循环。支持 `--tray` 启动参数（用于开机启动并最小化到托盘）。

## 状态卡片

主界面显示：

- 当前模式；
- NVIDIA 独显实时功耗（若能力可用）；
- 当前供电状态；
- CPU 最大状态；
- 屏幕亮度；
- 睡眠和关屏时间。

状态卡片优先显示当前供电分支实际生效的简洁数值；将鼠标悬停在卡片数值上可查看更完整信息。首次加载统一显示 `—`；读取失败时保留明确失败反馈。

关屏和睡眠时间按“关屏 / 睡眠”顺序显示，并尽量转换为自然语言（例如关屏 `600s`、睡眠 `0s` 显示为“10 分钟 / 永不”）。

状态刷新通过 `powercfg`、Windows 电池 API、系统性能计数和 `nvidia-smi` 获取。显卡功耗仅用于监控，PowerMode **不会**直接修改 NVIDIA 功耗限制。固件或驱动未暴露的项目会标明“不可用”，而不是把整次刷新一律判失败。

单次状态刷新设有超时保护；驱动或系统命令异常阻塞时，界面会恢复可操作并提示重试。

## 建议卡片

建议由本机逻辑计算（供电、配置的远程/高性能进程名、低电量阈值、温度保护是否已触发等），展示建议模式与原因。

- **顾问式**：只显示建议，**不会**在后台自动切换模式。
- **显式应用**：只有点击「一键应用」后才会执行对应标准模式切换。
- 若建议模式已是当前成功模式，应用按钮会禁用并显示“当前模式”。
- 供电等信息尚未完整时，界面会标明检测中 / 信息不完整，并给出保守建议。

默认远程进程示例：`Hermes,uu,SunloginClient,ToDesk`；高性能进程示例：`steam,GameBar,obs64`（可在功能中心修改）。

## 预设模式切换

| 模式 | 默认 CPU 上限 | 亮度 | 关屏 | 睡眠/休眠 | 适用场景 |
| --- | ---: | ---: | ---: | --- | --- |
| 远程推荐 | 32% | 50% | 60 秒 | 禁用 | Hermes、uu 远程、长期待机 |
| 低功耗 | 30% | 50% | 60 秒 | 禁用 | 轻负载、省电和低温 |
| 平衡 | 100% | 100% | 保留方案现有值 | 禁用 | 日常桌面 |
| 高性能 | 100% | 100% | 保留方案现有值 | 禁用 | 游戏、渲染与高负载 |

点击模式后，程序优先通过 `powercfg /setactive` 激活核心 Windows 电源方案，再在后台完成 CPU、亮度、睡眠等细项。主界面会立刻更新且不会锁死模式按钮；连续选择时以最后一次为准。

模式按钮旁显示 `1–4` 快捷键提示。切换到远程或低功耗时，关屏卡片会预览“1 分钟 / 永不”一类预期；后台完成后会重新读取方案并校准卡片。

> 实际支持情况取决于 Windows 版本、设备固件和驱动。部分台式机或外接显示器不支持通过 `powercfg` 调整亮度。

## 自定义 CPU 与自定义预设

### 远程自定义 CPU（专业模式）

使用滑杆或数字框设置 `20–50%` 的 CPU 上限，再点击「应用远程自定义 CPU」。输入不合法时应用按钮会禁用。专业模式中「远程 + 关闭 WiFi」会使用当前自定义 CPU 值，并在执行前确认。

滑杆初始默认值为 28%（仅影响自定义应用，不等于远程预设按钮的 32% 默认）。

### 自定义预设（功能中心）

在「功能中心」创建和管理自定义预设，可设置：

- 预设名称；
- 插电时的 CPU 最大状态、亮度和关屏时间；
- 电池供电时可选独立的 CPU、亮度和关屏；
- CPU 最低状态；
- 是否关闭 CPU 睿频。

自定义预设基于 Windows 节能方案，并保持睡眠和休眠关闭，以降低远程意外断开概率。若应用过程中任意关键命令失败，程序会尝试恢复切换前的 Windows 电源方案。

## 命令行使用

CLI 与 GUI 共用 `PowerModeSwitcher.bat`（便携包根目录或源码 `src\PowerMode.Cli\`；仓库根目录有兼容转发入口）。

在 PowerShell 或命令提示符中进入对应目录后：

### 查看状态

```bat
.\PowerModeSwitcher.bat status
```

### 校验当前模式

```bat
.\PowerModeSwitcher.bat verify
```

### 远程推荐

```bat
.\PowerModeSwitcher.bat remote
.\PowerModeSwitcher.bat remote 32
.\PowerModeSwitcher.bat remote 32 nowifi
```

### 低功耗

```bat
.\PowerModeSwitcher.bat saver
.\PowerModeSwitcher.bat saver 30
```

### 平衡与高性能

```bat
.\PowerModeSwitcher.bat balanced
.\PowerModeSwitcher.bat high
```

### 快捷别名

| 别名 | 模式 |
| --- | --- |
| `low` | 低功耗 |
| `normal` | 平衡 |
| `perf` | 高性能 |

### 语言

```bat
.\PowerModeSwitcher.bat lang zh
.\PowerModeSwitcher.bat lang en
```

不传参数时显示交互式菜单。GUI 与 CLI 均支持中文与 English；语言偏好保存在用户配置中。

## 自动化与保护

自动化位于「功能中心 → 自动化与安全」。**默认关闭**，避免首次运行意外改变当前模式。

若未创建自定义规则，启用后使用默认优先级：

1. 检测到远程程序 → 远程模式；
2. 检测到指定高性能程序 → 高性能模式；
3. 使用电池供电 → 低功耗模式；
4. 其他插电状态 → 平衡模式。

进程名支持逗号分隔，可带或不带 `.exe`。

### 可视化规则

「功能中心 → 规则」可添加、停用和删除规则。按优先级从高到低匹配，条件包括：

- 供电来源（交流电或电池）；
- 电池电量阈值；
- 时间段（支持 `22:00-07:30` 跨午夜）；
- 一个或多个运行中进程；
- 用户空闲秒数；
- 前台是否全屏；
- 是否处于远程桌面会话。

目标模式可选远程、低功耗、平衡或高性能。自动切换会记录触发规则、原因、耗时和成功状态。

### 低电量与温度保护

- 使用电池且电量达到设定阈值时，可自动切换低功耗并发送低电量提醒；
- 温度达到上限时进入低功耗；
- 温度降到恢复阈值后，回到保护前的模式；
- 关闭温度保护会清除保护锁定，不阻止后续自动规则。

温度采用驱动或固件实际暴露的 GPU / 系统热区数据；无可用传感器时不会误触发。

### 实时监控

启用后按配置间隔自动刷新硬件与电源状态，最低间隔 10 秒。窗口隐藏到托盘时，自动模式切换仍可运行；完整状态刷新主要在窗口可见时进行，以减少后台开销。

## 系统洞察与诊断

主窗口「洞察」（专业模式入口）可查看：

- CPU 负载与当前频率；
- NVIDIA GPU 实时功耗、利用率与温度（能力允许时）；
- 电量、充放电状态与预计剩余时间；
- 电池设计容量、满充容量、健康度与循环次数；
- 最近采样的 GPU 功耗 / 温度趋势；
- 模式切换历史（触发来源、规则、原因、耗时与错误）。

支持将遥测历史导出为 UTF-8 CSV，以及调用 Windows 生成标准 HTML 电池报告。

「导出诊断包」生成 ZIP，可包含配置、切换历史、当前会话日志、遥测 CSV 与只读系统信息。诊断包可能含设备型号与用户配置，分享前请检查内容。

## 恢复中心

主窗口「恢复」打开恢复中心，提供三类操作：

1. **撤销最近模式切换**  
   在切换历史中查找最近一次可撤销的成功标准模式切换，并切回其 `PreviousMode`。仅覆盖标准四模式之间的操作。

2. **恢复配置备份**  
   先为当前 `settings.json` 创建安全备份，再原子恢复最近一份**内容哈希不同**的备份。  
   **不改变**当前 Windows 活动电源方案。

3. **重置为默认值**  
   先创建安全备份，再将应用设置写回默认（含默认 **简单模式** 等）。  
   **历史、备份与遥测保留**；**不会**批量重置 Windows 电源方案。

危险操作前有确认对话框；结果区会反馈成功、部分成功或失败。

## 系统与维护

「功能中心 → 系统与维护」还包括：

- 模式切换托盘通知；
- 可选的手动切换预览；
- 当前用户开机启动及启动时隐藏到托盘；
- 启动后恢复上次模式；
- 切换历史开关；
- 可选的 GitHub Releases API 更新**检查**（只查询，不自动下载或安装；需自行配置 API 地址）；
- 配置版本保留数量、立即备份与一键恢复上一不同版本；
- 电脑厂商与充电上限能力提示（仅识别厂商并指向官方/BIOS 路径，不调用未公开固件接口）。

Windows 没有统一的充电上限 API。PowerMode 不代改 OEM 充电阈值。

### 专业模式实时日志

- 命令输出实时显示；
- 主题自适应背景与清晰字体；
- 命令、标准输出、错误与退出状态按前缀区分；
- 显示日志行数，可关闭自动滚动；
- 支持复制与清空；
- 过长日志会裁剪，避免长期占用过多内存。

## 快捷键与托盘

### 窗口内快捷键

| 快捷键 | 功能 |
| --- | --- |
| `1` | 远程推荐 |
| `2` | 低功耗 |
| `3` | 平衡 |
| `4` | 高性能 |
| `F5` | 刷新状态 |

在 CPU 数字框中输入时，`1–4` 不会触发模式切换。

### 全局快捷键

| 快捷键 | 功能 |
| --- | --- |
| `Ctrl + Alt + 1` | 远程推荐 |
| `Ctrl + Alt + 2` | 低功耗 |
| `Ctrl + Alt + 3` | 平衡 |
| `Ctrl + Alt + 4` | 高性能 |

若快捷键已被其他软件注册，Windows 可能无法分配给 PowerMode。

### 系统托盘

最小化主窗口后可隐藏到系统托盘。托盘右键菜单支持：

- 切换四个预设模式；
- 重新显示主窗口；
- 完全退出 PowerMode。

双击托盘图标可恢复主窗口。

主窗口会记住上次关闭时的大小、位置与最大化状态，并在显示器/DPI 变化后限制在可用工作区内。

## 配置、备份与数据位置

功能配置：

```text
%LOCALAPPDATA%\PowerMode\settings.json
```

主要包括：体验模式、自定义预设、AC/DC 参数、自动规则、低电量与温度保护、实时监控开关与间隔、启动/通知/历史/退出恢复选项、远程与高性能进程列表、更新检查地址与备份保留数量等。

功能中心支持将配置导出为 JSON，也可从 JSON 导入。

GUI 为加速执行，会将 BAT 中的 PowerShell 负载缓存到：

```text
%LOCALAPPDATA%\PowerMode\Cache
```

BAT 更新后会生成新版本缓存，通常无需手动清理。

其他位置：

```text
%LOCALAPPDATA%\PowerMode\switch-history.jsonl
%LOCALAPPDATA%\PowerMode\backups\
```

配置变更时可保留带 SHA-256 校验的版本快照。恢复会跳过与当前内容相同的最新快照，选择最近一个真正不同的版本，并在覆盖前再次保存安全备份。

## 安全与失败回滚

### 模式切换失败回滚

每次 GUI 模式切换前会记录当前活动电源方案。若后台设置失败，会重新激活之前的方案，并在状态栏与日志中显示错误。

成功与失败的切换均可写入 JSONL 历史。手动、托盘、快捷键、启动恢复、低电量、温度保护、建议应用与自动规则使用不同触发来源，便于排查。

### 退出时恢复 Windows 方案

可在功能中心启用「退出 PowerMode 时恢复启动前的电源方案」。启用后，关闭程序时会重新激活启动前记录的方案。这与恢复中心「重置应用设置」是不同能力。

### WiFi 安全提示

「远程 + 关闭 WiFi」可能立即断开当前远程会话。使用前请确认：

- 网线已连接；
- 远程软件确实通过有线网络通信；
- 必要时有人可在电脑旁恢复网络。

若无线网卡被关闭，可点击「恢复 WiFi」。Windows 可能要求管理员授权。

## 常见问题

### 简单模式里找不到功能中心或日志？

切换到专业模式即可。简单模式刻意收敛为状态、建议与四模式切换。

### 建议会自动改电源模式吗？

不会。必须点击「一键应用」才会切换。

### 重置默认会不会把 Windows 电源方案全部还原？

不会。重置只重写 PowerMode 的 `settings.json`（先安全备份），不批量重置 Windows 电源方案。

### 模式切换还需要等待吗？

不需要等待后台同步完成后才能继续选择。核心方案先切换，细项随后同步；连续选择以最后一次为准。

### 显卡功耗显示“不可用”

在终端执行：

```powershell
nvidia-smi --query-gpu=power.draw --format=csv,noheader,nounits
```

若命令不存在或报错，请更新 NVIDIA 驱动。AMD、Intel 或没有 NVIDIA 独显时不会提供该 GPU 功耗；其他指标仍可刷新。

### 点击刷新需要几秒

完整刷新会读取多个 `powercfg` 设置。硬件遥测在独立任务中采样，不阻塞模式按钮；也可打开「洞察」查看数据。

### 亮度没有立即变化

部分设备、外接显示器或驱动不支持通过电源方案修改实际亮度。PowerMode 仍会写入方案中的亮度值，硬件可能不响应。

### WiFi 无法关闭或恢复

网络适配器启停通常需要管理员权限。请确认 UAC 已允许，并检查无线网卡名称是否包含 `Wi`、`Wireless`、`Wi-Fi` 或 `WLAN`。

### 全局快捷键无效

可能被其他程序占用。关闭占用方并重启 PowerMode 以重新注册。

### 日志出现乱码

PowerMode 对 PowerShell 使用 UTF-8，对 Windows 原生命令使用当前 OEM code page。请确认运行最新便携版；重复启动只会唤醒同一实例。

### 配置损坏或想撤销修改

优先使用「恢复中心」或「系统与维护 → 恢复最近备份」。恢复前会创建安全版本。

若配置已无法加载，可在备份后删除：

```text
%LOCALAPPDATA%\PowerMode\settings.json
```

下次启动会使用默认配置（默认简单模式等）。

## 使用建议

- 长时间远程连接：优先「远程推荐」，或阅读建议后手动应用。
- 轻负载与低温：使用「低功耗」。
- 日常办公：使用「平衡」。
- 游戏或渲染：使用「高性能」。
- 首次启用自动模式时，先检查进程名列表是否符合习惯。
- 远程环境关闭 WiFi 前，务必确认有线网络正常。
