# PowerMode

PowerMode 是一个面向 Windows 笔记本和远程使用场景的电源模式切换工具，尤其适合通过 Hermes、uu 远程、ToDesk、向日葵等软件长期连接电脑的情况。

项目同时提供：

- WinUI 3 图形界面；
- BAT + PowerShell 命令行工具；
- 远程、低功耗、平衡和高性能预设；
- 自定义电源预设；
- 可视化规则引擎、低电量策略和温度保护；
- 实时 CPU / NVIDIA GPU / 电池遥测与趋势图；
- 切换历史、CSV、电池报告和诊断包导出；
- 托盘菜单、全局快捷键、开机启动和单实例保护；
- 模式校验、失败回滚、配置版本备份和更新检查。

当前桌面程序使用 **WinUI 3 + .NET 10 + Windows App SDK 2.2**，并以 x64 自包含便携版形式构建，不需要 MSIX 安装。

## 主要功能

### 快速模式切换

点击模式后，程序会优先通过 `powercfg /setactive` 激活核心 Windows 电源方案，再在后台完成 CPU、亮度、睡眠等细项设置。

主界面会立刻更新且不会锁死模式按钮。连续选择不同模式时，旧的后台同步会被取消，并以最后一次选择为准，避免配置互相覆盖。

### 状态面板

主界面显示：

- 当前模式；
- NVIDIA 独显实时功耗；
- 当前供电状态；
- CPU 最大状态；
- 屏幕亮度；
- 睡眠和关屏时间。

状态卡片优先显示当前供电分支实际生效的简洁数值，避免 AC/DC 原始信息挤压或截断；将鼠标悬停在卡片数值上，可查看完整原始信息。首次加载统一显示 `—`，读取失败时会保留明确的失败反馈，不会误报刷新成功。

关屏和睡眠时间会按“关屏 / 睡眠”的顺序显示并自动转换为自然语言，例如关屏 `600s`、睡眠 `0s` 会显示为“10 分钟 / 永不”。日志在中文界面使用微软雅黑 UI，英文界面使用 Cascadia Mono，减少高 DPI 下的中英文混排发虚。

状态刷新通过 `powercfg`、Windows 电池 API、系统性能计数和 `nvidia-smi` 获取。显卡功耗仅用于监控，PowerMode 不会直接修改 NVIDIA 显卡功耗限制。固件或驱动没有暴露的项目会明确显示“不可用”，而不是把整次刷新判定为失败。

### 实时日志

- 命令输出会实时显示，不需要等待整个操作结束；
- 使用主题自适应高对比度背景和清晰的等宽字体；
- 命令、标准输出、错误和退出状态采用不同前缀并按会话分组；
- 显示当前日志行数，可关闭自动滚动以便选择旧内容；
- 支持一键复制和清空；
- 每条命令包含执行时间；
- 状态栏显示成功、失败和总耗时；
- 日志过长时会自动裁剪，避免长期运行占用过多内存。

### 中英文界面

GUI 和命令行均支持中文与 English。语言偏好会保存到当前用户配置中。

## 预设模式

| 模式 | 默认 CPU 上限 | 亮度 | 关屏 | 睡眠/休眠 | 适用场景 |
| --- | ---: | ---: | ---: | --- | --- |
| 远程推荐 | 32% | 50% | 60 秒 | 禁用 | Hermes、uu 远程、长期待机 |
| 低功耗 | 30% | 50% | 60 秒 | 禁用 | 轻负载、省电和低温运行 |
| 平衡 | 100% | 100% | 保留方案现有值 | 禁用 | 日常桌面使用 |
| 高性能 | 100% | 100% | 保留方案现有值 | 禁用 | 游戏、渲染和高负载程序 |

远程和低功耗模式的 CPU 上限可在 `20–50%` 范围内调整。

> 实际支持情况取决于 Windows 版本、设备固件和驱动。部分台式机或外接显示器不支持通过 `powercfg` 调整亮度。

## 系统要求

- Windows 10 2004 或更高版本、Windows 11；
- x64 处理器；
- 开发构建需要 .NET 10 SDK，普通使用不需要单独安装 .NET 或 Windows App SDK 运行库；
- Windows PowerShell 5.1 或更高版本；
- 部分网络适配器操作需要管理员权限；
- NVIDIA 显卡功耗监控需要可用的 `nvidia-smi.exe`。

## 快速开始

### 启动图形界面

便携版用户双击解压目录中醒目的：

```text
00-START PowerMode.bat
```

从源码运行时，双击项目根目录中的：

```text
PowerModeWinUI.bat
```

启动器会优先运行 `dist` 中已经生成的便携版。源码、CLI、清单或 README 更新后，它会自动重新发布再启动。

### 启动命令行菜单

双击：

```text
PowerModeSwitcher.bat
```

不传参数时会显示交互式命令行菜单。

## 图形界面使用

主界面分为四个区域：

1. 顶部操作区：刷新、功能中心和语言切换；
2. 状态卡片：当前模式、功耗、供电、CPU、亮度和睡眠；
3. 模式控制区：预设模式、自定义 CPU 和高级操作；
4. 日志与状态栏：实时命令输出、执行进度和错误信息。

窗口缩窄时，顶部操作区会自动切换为紧凑图标模式，悬停仍可查看完整功能说明；恢复到宽窗口后，按钮文字会自动显示。

刷新状态时，刷新按钮会立即切换为进度指示，完成后自动恢复；日志复制和清空操作带有图标、禁用状态与悬停说明。

单次状态刷新设有 12 秒超时保护。驱动或系统命令异常阻塞时，界面会恢复可操作状态并提示重试，不会无限停留在加载中。

主窗口会记住上次关闭时的大小、位置和最大化状态。显示器断开、分辨率或 DPI 改变后，窗口会自动限制在当前可用工作区内，避免启动到屏幕外。

### 模式按钮

当前模式会保持蓝色选中状态。模式切换后，状态卡片会立即更新，详细设置继续在后台完成。

模式按钮右侧显示 `1–4` 快捷键提示。点击或按下快捷键后，底部会立即显示目标模式和后台同步进度；新的切换请求可以直接替代仍在同步的旧请求。

切换到远程推荐或低功耗时，关屏卡片会立即预览“1 分钟 / 永不”；切换到平衡或高性能时先显示“正在同步…”。后台设置完成后，PowerMode 会自动重新读取当前电源方案，并用实际关屏和睡眠时间校准卡片，不需要手动点击刷新。

### 自定义远程 CPU

可以使用滑杆或数字输入框设置 `20–50%` 的 CPU 上限，然后点击“应用远程自定义 CPU”。输入不合法时，应用按钮会自动禁用并显示提示。

### 高级操作

- **校验当前模式**：检查 CPU、亮度、睡眠、休眠和关屏设置；
- **一键修复**：重新应用当前模式，修复被 Windows 或其他软件改变的设置；
- **恢复 WiFi**：重新启用无线网卡，可能触发 UAC 管理员确认；
- **远程 + 关闭 WiFi**：仅适用于已经连接网线、且远程连接不依赖 WiFi 的情况。

在 PowerShell 或命令提示符中进入项目目录，然后使用以下命令。

```

### 校验当前模式

.\PowerModeSwitcher.bat verify
```

### 远程推荐模式

```bat
.\PowerModeSwitcher.bat remote
```

指定 CPU 上限：

```bat
.\PowerModeSwitcher.bat remote 32
```

插网线后同时关闭 WiFi：

```bat
.\PowerModeSwitcher.bat remote 32 nowifi
```

### 低功耗模式

```bat
.\PowerModeSwitcher.bat saver
.\PowerModeSwitcher.bat saver 30
```

### 平衡与高性能

```bat
.\PowerModeSwitcher.bat balanced
.\PowerModeSwitcher.bat high
```

### 快捷别名

```bat
.\PowerModeSwitcher.bat low

.\PowerModeSwitcher.bat perf
```

对应关系：

| 别名 | 模式 |
| --- | --- |
| `low` | 低功耗 |
| `normal` | 平衡 |
| `perf` | 高性能 |

```bat

```

## 自定义预设

点击主界面右上角的“功能中心”，可以创建和管理自定义预设。

每个预设可以设置：

- 预设名称；
- 插电时的 CPU 最大状态、亮度和关屏时间；
- 电池供电时可使用独立的 CPU、亮度和关屏参数；
- CPU 最低状态；
- 是否关闭 CPU 睿频。

自定义预设基于 Windows 节能方案，并保持睡眠和休眠关闭，以降低远程连接意外断开的概率。

如果应用自定义预设时任意关键命令失败，程序会尝试恢复切换前的 Windows 电源方案。

## 自动化功能

自动化功能位于“功能中心 → 自动化与安全”。默认关闭，避免首次运行时意外改变当前模式。

如果没有创建自定义规则，启用后使用以下默认优先级：

1. 检测到远程程序：切换到远程模式；
2. 检测到指定高性能程序：切换到高性能模式；
3. 使用电池供电：切换到低功耗模式；
4. 其他插电状态：切换到平衡模式。

远程程序和高性能程序均可使用逗号分隔的进程名进行配置，例如：

```text
Hermes,uu,SunloginClient,ToDesk
```

```text
steam,GameBar,obs64
```

进程名可以填写带或不带 `.exe` 的形式。

### 可视化规则

“功能中心 → 规则”可以添加、停用和删除规则。规则按优先级从高到低匹配，当前支持：

- 供电来源：交流电或电池；
- 电池电量阈值；
- 时间段，支持 `22:00-07:30` 这类跨午夜范围；
- 一个或多个运行中的进程；
- 用户空闲秒数；
- 前台程序是否全屏；
- 是否处于远程桌面会话。

目标模式可以选择远程、低功耗、平衡或高性能。每次自动切换都会记录触发规则、原因、耗时和成功状态。

### 低电量与温度保护

- 使用电池且电量达到设定阈值时，可自动切换低功耗并发送低电量提醒；
- 温度达到上限时立即进入低功耗；
- 温度下降到恢复阈值后，自动回到保护前的模式；
- 关闭温度保护会立即清除保护锁定，不会阻止后续自动规则。

温度采用驱动或固件实际暴露的 GPU / 系统热区数据；没有可用温度传感器时不会误触发。

### 实时监控

启用实时状态监控后，PowerMode 会按照配置的时间间隔自动刷新硬件和电源状态。最低间隔为 10 秒。

窗口隐藏到托盘时，自动模式切换仍会运行；完整状态刷新只在窗口可见时进行，以减少后台开销。

## 系统洞察

点击主窗口顶部“洞察”，可以查看：

- CPU 负载与当前频率；
- NVIDIA GPU 实时功耗、利用率和温度；
- 当前电量、充放电状态和预计剩余时间；
- 电池设计容量、满充容量、健康度与循环次数；
- 最近采样的 GPU 功耗 / 温度双折线趋势；
- 模式切换历史，包括触发来源、规则、原因、耗时与错误。

洞察页支持把遥测历史导出为 UTF-8 CSV，也可调用 Windows 生成标准 HTML 电池报告。

## 系统与维护

“功能中心 → 系统与维护”包括：

- 模式切换托盘通知；
- 可选的手动切换预览；
- 当前用户开机启动及随 Windows 启动时隐藏到托盘；
- 启动后恢复上次模式；
- 切换历史开关；
- GitHub Releases API 更新检查（只查询，不自动下载或安装）；
- 配置版本保留数量、立即备份和一键恢复上一不同版本；
- 电脑厂商与充电上限能力提示。

Windows 没有统一的充电上限 API。PowerMode 只识别 Lenovo、ASUS、Dell、HP 等厂商并提供官方工具或 BIOS 路径，不调用未公开的固件接口。

“导出诊断包”会生成 ZIP，包含配置、切换历史、当前会话日志、遥测 CSV 和只读系统信息。诊断包可能包含设备型号、进程命令结果和用户配置，分享给他人前请先检查内容。

## 快捷键与托盘

### 窗口内快捷键

| 快捷键 | 功能 |
| --- | --- |
| `1` | 远程推荐 |
| `2` | 低功耗 |
| `3` | 平衡 |
| `4` | 高性能 |
| `F5` | 刷新状态 |

在 CPU 输入框中输入数字时，`1–4` 不会触发模式切换。

### 全局快捷键

即使 PowerMode 窗口没有获得焦点，也可以使用：

| 快捷键 | 功能 |
| --- | --- |
| `Ctrl + Alt + 1` | 远程推荐 |
| `Ctrl + Alt + 2` | 低功耗 |
| `Ctrl + Alt + 3` | 平衡 |
| `Ctrl + Alt + 4` | 高性能 |

如果快捷键已被其他软件注册，Windows 可能无法将其分配给 PowerMode。

### 系统托盘

最小化主窗口后，PowerMode 会隐藏到系统托盘。托盘右键菜单支持：

- 切换四个预设模式；
- 重新显示主窗口；
- 完全退出 PowerMode。

双击托盘图标可以恢复主窗口。

PowerMode 只允许一个实例运行。重复启动便携版时会唤醒已有窗口，不会创建第二套托盘、热键或自动化循环。

## 安全与恢复

### 切换失败回滚

每次模式切换前，GUI 会记录当前活动电源方案。如果后台设置失败，会重新激活之前的方案，并在状态栏和日志中显示错误。

成功与失败的切换均可写入 JSONL 历史。手动、托盘、快捷键、启动恢复、低电量、温度保护和自动规则会使用不同的触发来源，便于排查异常切换。

### 配置版本恢复

每次配置发生变化时会保留带 SHA-256 校验值的版本快照。恢复功能会跳过与当前配置内容相同的最新快照，选择最近一个真正不同的版本，并在覆盖前再次保存安全备份。

### 退出时恢复

可以在功能中心启用“退出 PowerMode 时恢复启动前的电源方案”。启用后，程序关闭时会重新激活启动 PowerMode 前使用的方案。

### WiFi 安全提示

“远程 + 关闭 WiFi”可能立即断开当前远程会话。使用前请确认：

- 网线已经连接；
- 远程软件确实通过有线网络通信；
- 当前电脑旁有人可以在必要时恢复网络。

如果无线网卡被关闭，可以点击“恢复 WiFi”。Windows 可能要求管理员授权。

## 配置文件

PowerMode 的功能配置保存在：

```text
%LOCALAPPDATA%\PowerMode\settings.json
```

配置内容包括：

- 自定义预设；
- AC / DC 独立参数；
- 自动规则、低电量与温度保护；
- 实时监控开关和间隔；
- 启动、通知、历史和退出恢复选项；
- 远程和高性能程序列表；
- 更新检查地址与备份保留数量。

功能中心支持将配置导出为 JSON，也可以从 JSON 文件导入。

GUI 为了加快命令执行，会把 BAT 中的 PowerShell 负载缓存到：

```text
%LOCALAPPDATA%\PowerMode\Cache
```

BAT 文件更新后会自动生成新版本缓存，不需要手动清理。

其他数据位置：

```text
%LOCALAPPDATA%\PowerMode\switch-history.jsonl
%LOCALAPPDATA%\PowerMode\backups\
```

## 构建便携版

双击根目录的：

```text
Build-Portable.bat
```

或执行：

```powershell
.\scripts\Publish-Portable.ps1 -Root . -CreateZip
```

完整便携版位于：

```text
dist\PowerMode-win-x64\
dist\PowerMode-win-x64.zip
```

构建脚本会先发布到暂存目录，成功后再替换旧版本，同时刷新 ZIP。复制整个目录或解压 ZIP 即可使用。根目录只保留启动入口、命令行工具和说明文档，运行库统一放在 `App` 子目录中。目标电脑不需要另装运行库；请双击排在最前面的 `00-START PowerMode.bat`，不要单独移动 `App\PowerMode.exe`。

## 从源码构建

### 使用命令行

```powershell
dotnet build .\PowerMode.slnx -c Release -p:Platform=x64
```

`dotnet build` 用于开发验证；分发给其他电脑时请使用上一节的 `dotnet publish --self-contained true`。

构建产物位于：

```text
src\PowerMode.App\bin\Release\net10.0-windows10.0.26100.0\win-x64\
```

启动程序：

```powershell
.\src\PowerMode.App\bin\Release\net10.0-windows10.0.26100.0\win-x64\PowerMode.exe
```

### 清理并重新构建

```powershell
dotnet clean .\src\PowerMode.App\PowerMode.App.csproj -c Release -p:Platform=x64
dotnet build .\src\PowerMode.App\PowerMode.App.csproj -c Release -p:Platform=x64
```

## 常见问题

### 模式切换还需要等待吗

不需要等待后台同步后才能继续选择。核心 Windows 电源方案会先立即切换，CPU、亮度、睡眠等细项随后同步；模式按钮保持可用。连续选择时，以最后一次选择为准。

### 显卡功耗显示“不可用”

请在终端执行：

```powershell
nvidia-smi --query-gpu=power.draw --format=csv,noheader,nounits
```

如果命令不存在或报错，请更新 NVIDIA 驱动。AMD、Intel 或没有 NVIDIA 独显的电脑目前不会提供 GPU 功耗；CPU、电池等其他指标仍会继续刷新。

### 点击刷新需要几秒

完整刷新会读取多个 `powercfg` 设置，因此比单纯切换电源方案慢。硬件遥测在独立后台任务中采样，不会阻塞模式按钮；也可以直接打开“洞察”查看最新数据。

### 亮度没有立即变化

部分设备、外接显示器或显卡驱动不支持通过 Windows 电源方案修改实际亮度。PowerMode 仍会写入电源方案中的亮度值，但硬件可能不会响应。

### WiFi 无法关闭或恢复

网络适配器启用和禁用通常需要管理员权限。请确认 UAC 请求已允许，并检查无线网卡名称是否包含 `Wi`、`Wireless`、`Wi-Fi` 或 `WLAN`。

### 全局快捷键无效

可能是快捷键已被其他程序占用。关闭占用快捷键的软件并重新启动 PowerMode，使程序重新注册快捷键。

### 日志出现乱码

PowerMode 对 PowerShell 使用 UTF-8，对 Windows 原生命令使用当前 OEM code page。请确认正在运行最新便携版；重复启动只会唤醒同一实例。

### 功能中心的配置损坏或想撤销修改

优先使用“系统与维护 → 恢复最近备份”。恢复前会自动创建安全版本。

如果配置已经无法加载，也可以备份后删除：

```text
%LOCALAPPDATA%\PowerMode\settings.json
```

下次启动时会自动使用默认配置。

## 使用建议

- 长时间远程连接：优先使用“远程推荐”；
- 轻负载和低温需求：使用“低功耗”；
- 日常办公：使用“平衡”；
- 游戏或渲染：使用“高性能”；
- 首次启用自动模式时，先检查进程名列表是否符合自己的使用习惯；
- 在远程环境中关闭 WiFi 前，务必确认有线网络工作正常。
